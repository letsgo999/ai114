2. 실제 발생한 문제와 해결책

중요!

이 섹션은 실제 개발 과정에서 발생한 문제와 해결책입니다. 다음 프로젝트에서 동일한 문제가 발생하지 않도록 반드시 숙지하세요.

문제 1: 휴지통 기능 누락

발생 상황

"관리자 모드에서 보고서 레코드 별 뒷부분에 휴지통 아이콘을 추가해서 삭제 처리하면 휴지통으로 넣어서 통계에 잡히지 않도록 하고, 30일 뒤에는 복원 불가능하게 영구 삭제 처리해줘."

해결책

  1. DB 마이그레이션으로 deleted_at 컬럼 추가
  2. 소프트 삭제 API 구현 (DELETE /api/tasks/:id)
  3. 복원 API 구현 (POST /api/tasks/:id/restore)
  4. 휴지통 목록 API 구현 (GET /api/admin/trash)
  5. 모든 조회 쿼리에 WHERE deleted_at IS NULL 조건 추가
예방책: 초기 DB 스키마에 deleted_at 컬럼 필수 포함!

문제 2: 삭제 버튼 미표시

발생 상황

삭제 기능 구현 후 배포했으나 버튼이 화면에 표시되지 않음

원인

  • 아이콘만 있는 버튼 (텍스트 없음)
  • 버튼 크기/색상이 눈에 띄지 않음
  • 브라우저 캐시 문제

❌ 잘못된 예

<button><i class="fas fa-trash"></i></button>

✅ 올바른 예

<button class="bg-red-100 text-red-600">
  <i class="fas fa-trash-alt mr-1"></i>삭제
</button>
예방책: 모든 버튼에 반드시 텍스트 레이블 포함 + 명확한 색상!

문제 3: 점수 표시 불명확

발생 상황

"분석 보고서 중 추천 AI도구 TOP 5 목록 뒷쪽 점수가 무엇을 뜻하는지 사용자들이 기준이나 의미, 점수를 이해하기 어려워."

해결책

  1. 점수를 100점 만점으로 정규화
  2. 점수 옆에 "(100점 만점)" 표시 추가
  3. 제목 옆에 (i) 인포 아이콘 추가
  4. 마우스 오버 시 툴팁으로 점수 산정 기준 표시

적용 결과

85점 (100점 만점) 점수 산정: 키워드 일치 40%, 평점 30%, 난이도 20%, 가격 10%
예방책: 점수/평점 표시 시 만점 기준 + 인포 아이콘 툴팁 필수!

  • • 난이도 (20%)

16. 최종 점검 체크리스트

개발 완료 전 필수 확인!

아래 항목을 모두 확인한 후에만 프로덕션 배포를 진행하세요.

데이터베이스

  • 모든 테이블에 deleted_at 컬럼 있음
  • deleted_at 인덱스 생성됨
  • 모든 조회 쿼리에 deleted_at IS NULL 조건

API

  • GET /api/admin/trash (휴지통 목록)

UI - 버튼 ⚠️

  • 삭제 버튼에 "삭제" 텍스트 포함
  • 삭제 버튼 빨간색 스타일 적용

UI - 점수 ⚠️

  • 점수 XX점 (100점 만점) 형식
  • 인포 아이콘 있음
